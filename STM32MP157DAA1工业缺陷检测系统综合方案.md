# STM32MP157DAA1 工业缺陷检测与分拣系统综合方案

## 1. 项目定位

本方案面向嵌入式芯片与系统设计竞赛，目标是基于 `STM32MP157DAA1 + STM32F4 + 六自由度总线舵机机械臂 + UVC 摄像头 + 背光 + 涡流模块 + 7寸1024x600 RGB屏`，实现一个可落地、可演示、可答辩的工业零件缺陷检测与自动分拣系统。

系统重点不放在“堆模型”，而放在以下三点：

| 核心目标 | 说明 |
|---|---|
| 多模态检测 | 表面视觉、背光轮廓、涡流近表层检测联合判定 |
| 稳定闭环 | 先视觉定位到检测中心，再触发高质量检测，再执行分拣 |
| 可追溯管理 | 7寸屏实时显示，本地记录，云端存档，支持 AI 复核与人工复核 |

---

## 2. 推荐检测对象

结合你的板子算力、机械结构、背光方案、涡流模块和比赛稳定性，最推荐的零件不是螺丝，而是**薄型金属件**。

| 优先级 | 零件类型 | 推荐原因 | 可检测缺陷 |
|---|---|---|---|
| 1 | 平垫圈、异形垫片 | 平面度高、便于背光检测、容易居中 | 外径异常、内孔偏差、缺口、裂纹、毛刺、锈蚀、变形 |
| 2 | 弹簧垫圈、齿形垫圈 | 特征明显、好做分类与良坏判断 | 开口异常、断裂、崩边、锈蚀、变形 |
| 3 | 薄型冲压连接片、端子片 | 工业感强，适合展示多缺陷 | 孔位偏差、冲压裂纹、弯折、毛刺、压伤 |

推荐最终答辩口径：

> 本系统面向薄型金属零件，采用表面视觉、透射背光和近表层涡流检测相结合的多模态缺陷检测方案，实现自动检测、自动分拣和历史数据追溯。

### 3.3 推荐系统流程

```text
上电自检
  -> 输送带运行
  -> 光电/视觉发现零件进入检测区
  -> 视觉实时计算零件中心偏差
  -> F4 根据偏差做低速微调
  -> 零件稳定停在摄像头中心
  -> MP157 采集表面图 + 基于常亮背光的轮廓图
  -> F4 同步采集涡流特征
  -> MP157 执行检测与融合判断
  -> MP157 下发 SORT_GOOD / SORT_BAD
  -> F4 控制机械臂分拣
  -> MP157 保存记录并上传云端
  -> 进入下一件循环
```

## 4. 板端算力适配的模型清单

`STM32MP157DAA1` 有双核 `Cortex-A7` 和 GPU，但**没有 NPU**，因此模型部署原则是：

| 原则 | 说明 |
|---|---|
| 优先轻量模型 | 避免大模型导致推理时间过长 |
| 优先 INT8 量化 | 板端主要靠 CPU 推理，量化非常重要 |
| 优先单任务模型 | 分类、检测、分割分开做，比全能模型更稳 |
| 能用规则法就先不用模型 | 背光轮廓、居中定位优先 OpenCV |

### 4.1 推荐模型

| 优先级 | 模型 | 任务 | 推荐程度 | 说明 |
|---|---|---|---|---|
| A | MobileNetV3-Small INT8 | 良坏分类 | 很高 | 作为主分类模型最合适 |
| A | MobileNetV1 0.5_128 INT8 | 零件分类/好坏二分类 | 高 | 轻量、稳定、适合做基线 |
| A | SSD MobileNetV1 300 INT8 | 目标粗定位 | 高 | 用于检测零件是否进入 ROI、粗框定位 |
| A | MobileNetV3-Small + LR-ASPP INT8 | 轻量分割 | 高 | 用于表面缺陷掩码与可视化 |
| B | MobileNetV2 0.35/0.5 INT8 | 分类/检测骨干 | 中 | 作为备用模型 |
| B | SSDLite + MobileNetV2 | 轻量检测 | 中 | 检测大缺陷或框选零件 |
| C | EfficientAD | 异常检测 | 云端推荐 | 不建议放板端实时主链路 |
| C | PatchCore | 异常检测 | 云端推荐 | 不建议放板端实时主链路 |
| C | PaDiM | 异常检测 | 云端推荐 | 不建议放板端实时主链路 |

### 4.2 最终推荐模型组合

| 子任务 | 推荐方案 |
|---|---|
| 零件进入画面、居中定位 | OpenCV 轮廓法，不上模型 |
| 零件类型识别 | MobileNetV3-Small INT8 |
| 表面良坏分类 | MobileNetV3-Small INT8 |
| 表面缺陷区域标注 | MobileNetV3-Small + LR-ASPP INT8 |
| 背光轮廓检测 | OpenCV 二值化 + 轮廓匹配 |
| 涡流异常判断 | F4 提取特征 + 阈值/规则法 |
| 云端复杂复核 | EfficientAD 或 PatchCore |

## 5. 为什么“先视觉居中，再启动检测”

如果零件没有稳定停在摄像头中心，后面所有检测都会受影响：

| 问题 | 影响 |
|---|---|
| 零件位置漂移 | 图像 ROI 不稳定，模型误判率上升 |
| 目标大小变化 | 缺陷尺度不一致，阈值难统一 |
| 背光轮廓偏移 | 孔位/轮廓比对误差大 |
| 机械臂取件偏移 | 分拣稳定性下降 |

因此本系统必须采用：

> **视觉定位闭环 -> 停在中心 -> 触发高质量检测 -> 执行分拣**

## 6. 视觉定位到摄像头中央的实现方案

### 6.1 核心思想

传送带只沿一个方向运动，因此视觉定位本质上是一个**单轴闭环定位问题**。

系统不是让相机找满世界的零件，而是通过机械结构和导轨把零件姿态先“管住”，再让相机只做以下三件事：

| 步骤 | 作用 |
|---|---|
| 发现零件进入视野 | 判断是否有零件进入检测窗口 |
| 计算零件中心偏差 | 计算零件中心点与图像中心的差值 |
| 输出偏差给 F4 | 让 F4 微调传送带直到零件居中 |

### 6.2 机械与光学前提

为了让视觉定位稳定，必须先做这些结构约束：

| 项目 | 建议 |
|---|---|
| 输送姿态 | 用窄导轨或限位槽保证零件大致沿同一方向进入 |
| 单件间隔 | 增加挡料机构或分料结构，避免同屏出现多个零件 |
| 相机安装 | 固定高度，固定焦距，固定角度，不做上下移动主方案 |
| 成像区域 | 只允许一个检测窗口，避免背景杂乱 |
| 光源 | 表面检测使用环形或条形补光；底部背光常亮，作为持续轮廓光源 |

### 6.3 坐标定义

定义相机图像坐标：

| 变量 | 含义 |
|---|---|
| `W` | 图像宽度 |
| `H` | 图像高度 |
| `Cx_ref = W / 2` | 图像中心 X 坐标 |
| `Cy_ref = H / 2` | 图像中心 Y 坐标 |
| `Cx_obj` | 当前零件中心 X 坐标 |
| `Cy_obj` | 当前零件中心 Y 坐标 |
| `dx = Cx_obj - Cx_ref` | 水平方向偏差 |
| `dy = Cy_obj - Cy_ref` | 垂直方向偏差 |

由于传送带主运动方向通常只有一个轴，因此实际控制主要使用 `dx` 或沿带方向的偏差量。

### 6.4 视觉定位流程

#### 方案一：推荐的低成本稳定方案

使用低分辨率预览流做定位，使用高质量单帧做检测：

| 阶段 | 分辨率建议 | 用途 |
|---|---|---|
| 预览定位 | `640x360` 或 `640x480` | 实时计算中心偏差 |
| 停稳检测 | `1280x720` 或相机原生分辨率 | 采集高质量表面图像 |
| 背光检测 | 基于常亮背光窗口抓拍单帧 | 轮廓/孔位/裂纹判断 |

#### 方案二：背光常亮时的成像建议

你的背光如果是**一直开启**，流程上不需要再单独“开背光”。系统只需要区分“定位预览帧”“表面检测帧”“背光轮廓帧”的用途即可。

| 图像类型 | 光照条件 | 建议做法 |
|---|---|---|
| 定位预览帧 | 背光常亮 | 直接用于轮廓提取和中心点计算 |
| 背光轮廓帧 | 背光常亮 | 直接抓拍并做二值轮廓分析 |
| 表面检测帧 | 背光常亮 + 上方补光 | 通过上方补光增强表面纹理，并尽量让上方光强占主导 |

如果后续发现背光常亮会干扰表面纹理细节，可做以下优化，但现阶段不作为必选项：

| 优化项 | 说明 |
|---|---|
| 遮光罩 | 减少底部漏光对表面图的影响 |
| 上方高亮补光 | 提高表面反射纹理对比度 |
| ROI 分区检测 | 表面图与轮廓图使用不同 ROI |
| 分时调光 | 后续再升级为表面拍照时降低背光亮度 |

#### 定位算法步骤

| 步骤 | 处理内容 |
|---|---|
| 1 | 读取预览图像 |
| 2 | 灰度化或转 HSV |
| 3 | 做 ROI 裁剪，只保留检测窗口 |
| 4 | 根据零件与背景差异做阈值分割 |
| 5 | 形态学开闭运算去噪 |
| 6 | 提取最大连通域或主轮廓 |
| 7 | 计算外接矩形中心点或轮廓质心 |
| 8 | 计算 `dx` 偏差并发送给 F4 |
| 9 | F4 低速微调，直到连续多帧满足中心容差 |

### 6.5 中心判定条件

建议不要只看单帧，而要看“连续稳定”。

| 条件 | 建议值 | 说明 |
|---|---|---|
| 粗定位阈值 | `|dx| < 40 px` | 进入低速微调区 |
| 精定位阈值 | `|dx| < 8 px` | 满足精确居中 |
| 稳定帧数 | `3 ~ 5` 帧 | 防止抖动误停 |
| 面积变化容差 | `< 5%` | 防止零件还在滑动 |
| 触发检测条件 | 精定位 + 连续稳定 + 输送带速度为 0 | 检测前必须彻底停稳 |

### 6.6 像素偏差换算成输送带位移

为了让 F4 能根据视觉偏差精准微调，必须做一次标定。

#### 标定方式

| 步骤 | 内容 |
|---|---|
| 1 | 让输送带移动固定距离，例如 `10 mm` |
| 2 | 记录图像中零件中心移动的像素值 `delta_px` |
| 3 | 计算比例系数 `k_mm_per_px = 10 / delta_px` |
| 4 | 再结合编码器脉冲数，得到 `k_pulse_per_px` |

#### 控制公式

| 公式 | 说明 |
|---|---|
| `move_mm = dx * k_mm_per_px` | 视觉偏差换算成实际位移 |
| `move_pulse = dx * k_pulse_per_px` | 视觉偏差换算成编码器脉冲 |

F4 不需要理解图像，只需要接收 MP157 发来的偏差值或目标脉冲增量。

### 6.7 推荐状态机

| 状态 | 含义 | 转移条件 |
|---|---|---|
| `IDLE` | 待机 | 点击开始 |
| `FEEDING` | 输送中 | 光电触发或视觉检测到零件 |
| `TRACKING` | 视觉粗定位 | 偏差进入微调区 |
| `CENTERING` | 低速精定位 | 连续多帧满足中心阈值 |
| `CAPTURE_READY` | 已停稳等待采图 | 表面图和常亮背光轮廓图采集成功 |
| `INSPECTING` | 检测中 | 推理和融合结束 |
| `SORTING` | 分拣中 | F4 完成动作 |
| `RECORDING` | 保存上传中 | 本地存储成功 |
| `ALARM` | 故障态 | 人工复位 |

### 6.8 推荐通信字段

#### MP157 -> F4

| 字段 | 类型 | 说明 |
|---|---|---|
| `cmd` | `uint8_t` | 命令码 |
| `track_valid` | `uint8_t` | 是否检测到目标 |
| `dx_px` | `int16_t` | 当前中心偏差像素值 |
| `target_pulse_delta` | `int32_t` | 建议微调脉冲数 |
| `inspect_start` | `uint8_t` | 是否开始检测 |
| `sort_type` | `uint8_t` | 0-无动作 1-良品 2-坏品 3-待复核 |

#### F4 -> MP157

| 字段 | 类型 | 说明 |
|---|---|---|
| `state` | `uint8_t` | 当前状态机状态 |
| `belt_speed` | `int16_t` | 输送带速度 |
| `encoder_count` | `int32_t` | 编码器累计值 |
| `part_present` | `uint8_t` | 光电是否检测到零件 |
| `eddy_amp` | `float` | 涡流幅值特征 |
| `eddy_phase` | `float` | 涡流相位特征 |
| `sort_done` | `uint8_t` | 分拣动作是否完成 |
| `alarm_code` | `uint16_t` | 故障码 |

### 6.9 推荐控制逻辑

```text
如果视觉未看到零件:
    F4 维持输送带低速运行

如果视觉看到零件:
    MP157 计算 dx
    如果 |dx| > 粗定位阈值:
        F4 中速调整
    如果 粗定位阈值 >= |dx| > 精定位阈值:
        F4 低速点动
    如果连续 N 帧 |dx| < 精定位阈值:
        F4 停带
        MP157 触发表面图采集
        MP157 抓拍常亮背光轮廓图
        F4 采集涡流数据
        MP157 执行检测与融合
```

### 6.10 建议的算法选型

视觉居中这一步**不要一开始就上深度学习**，直接用 OpenCV 更稳：

| 任务 | 推荐方法 |
|---|---|
| 是否有零件 | 连通域面积阈值 |
| 零件中心点 | 轮廓质心 / 最小外接矩形中心 |
| 零件姿态角 | 最小外接矩形角度 |
| 单件跟踪 | 最近中心点匹配 |
| 防止误停 | 连续帧稳定判定 |

### 6.11 MP157 ↔ F4 串口协议完整数据帧定义

这套协议按“**稳定、可扩展、容易抓包排查**”设计，推荐先用 `UART` 跑通，后续如果需要更强抗干扰再迁移到 `CAN`。

#### 6.11.1 物理层配置

| 项目 | 推荐值 |
|---|---|
| 接口 | UART |
| 波特率 | `921600`，调试初期可先用 `115200` |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | None |
| 字节序 | 小端 |
| CRC | CRC16-IBM / Modbus |

#### 6.11.2 通用帧结构

| 字段 | 类型 | 字节数 | 说明 |
|---|---|---|---|
| `sof1` | `uint8_t` | 1 | 固定为 `0x55` |
| `sof2` | `uint8_t` | 1 | 固定为 `0xAA` |
| `version` | `uint8_t` | 1 | 协议版本，当前为 `0x01` |
| `src` | `uint8_t` | 1 | 源设备 ID |
| `dst` | `uint8_t` | 1 | 目标设备 ID |
| `frame_type` | `uint8_t` | 1 | 帧类型 |
| `msg_id` | `uint8_t` | 1 | 具体消息 ID |
| `seq` | `uint16_t` | 2 | 帧序号，循环递增 |
| `payload_len` | `uint16_t` | 2 | 负载长度 |
| `payload` | `uint8_t[]` | N | 负载数据 |
| `crc16` | `uint16_t` | 2 | 从 `version` 到 `payload` 的 CRC |

推荐设备 ID：

| 设备 | ID |
|---|---|
| MP157 | `0xA1` |
| F4 | `0xF4` |
| 预留上位机/调试器 | `0xD0` |

推荐帧类型：

| `frame_type` | 含义 |
|---|---|
| `0x01` | 命令帧 `CMD` |
| `0x02` | 应答帧 `ACK` |
| `0x03` | 状态上报帧 `STATUS` |
| `0x04` | 心跳帧 `HEARTBEAT` |
| `0x05` | 告警帧 `ALARM` |
| `0x06` | 参数同步帧 `PARAM` |

#### 6.11.3 ACK 机制

| 项目 | 规则 |
|---|---|
| 需要 ACK 的消息 | 所有 `CMD`、`PARAM`、`SORT_REQUEST`、`RESET_ALARM` |
| ACK 超时 | `50 ms` |
| 最大重发次数 | `3` 次 |
| 失败处理 | MP157 进入告警页提示通信异常，F4 进入安全停机 |

ACK 负载格式：

| 字段 | 类型 | 说明 |
|---|---|---|
| `ack_msg_id` | `uint8_t` | 被应答的消息 ID |
| `ack_seq` | `uint16_t` | 被应答帧序号 |
| `result` | `uint8_t` | `0x00` 成功，非 0 为失败 |
| `error_code` | `uint16_t` | 错误码 |

#### 6.11.4 设计约束

推荐**不要在串口协议里直接发 `float`**，统一使用定点整数，调试和跨平台都更稳。

| 物理量 | 线上表示方式 |
|---|---|
| 速度 | `int16_t`，单位 `mm/s` |
| 像素偏差 | `int16_t`，单位 `px` |
| 位移脉冲 | `int32_t`，单位 `pulse` |
| 涡流幅值 | `int16_t`，实际值乘 `1000` |
| 涡流相位 | `int16_t`，实际值乘 `100` |
| 置信度 | `uint16_t`，范围 `0~1000` |

#### 6.11.5 MP157 -> F4 命令定义

| `msg_id` | 名称 | 作用 |
|---|---|---|
| `0x10` | `CMD_START_AUTO` | 启动自动流程 |
| `0x11` | `CMD_STOP_AUTO` | 停止自动流程 |
| `0x12` | `CMD_SET_BELT` | 设置传送带模式和目标速度 |
| `0x13` | `CMD_TRACK_OFFSET` | 下发视觉偏差，驱动微调 |
| `0x14` | `CMD_INSPECT_PREPARE` | 已居中，准备停带和锁定 |
| `0x15` | `CMD_SORT_REQUEST` | 请求分拣 |
| `0x16` | `CMD_MANUAL_AXIS` | 手动控制输送带或机械臂 |
| `0x17` | `CMD_SET_SURFACE_LIGHT` | 设置表面补光亮度 |
| `0x18` | `CMD_RESET_ALARM` | 清除告警 |
| `0x19` | `CMD_GO_HOME` | 机械臂回零/回待机位 |
| `0x1A` | `CMD_PARAM_SYNC` | 参数同步 |

##### `0x10 CMD_START_AUTO`

| 字段 | 类型 | 说明 |
|---|---|---|
| `work_mode` | `uint8_t` | `0`: 单步，`1`: 连续 |
| `part_type` | `uint8_t` | 当前零件类型编号 |
| `reserved` | `uint16_t` | 保留 |

##### `0x12 CMD_SET_BELT`

| 字段 | 类型 | 说明 |
|---|---|---|
| `belt_mode` | `uint8_t` | `0`: 停止，`1`: 低速，`2`: 中速，`3`: 点动 |
| `direction` | `uint8_t` | `0`: 反向，`1`: 正向 |
| `target_speed_mmps` | `int16_t` | 目标速度 |
| `jog_time_ms` | `uint16_t` | 点动时长 |

##### `0x13 CMD_TRACK_OFFSET`

这是定位闭环的核心消息，建议 `20~30 Hz` 下发。

| 字段 | 类型 | 说明 |
|---|---|---|
| `track_valid` | `uint8_t` | `0`: 未发现目标，`1`: 发现目标 |
| `center_locked` | `uint8_t` | `0`: 未锁定，`1`: 已锁定 |
| `dx_px` | `int16_t` | X 方向像素偏差 |
| `dy_px` | `int16_t` | Y 方向像素偏差 |
| `target_pulse_delta` | `int32_t` | 建议微调脉冲数 |
| `stable_count` | `uint8_t` | 连续稳定帧计数 |
| `control_level` | `uint8_t` | `0`: 继续粗调，`1`: 低速微调，`2`: 停带锁定 |

##### `0x14 CMD_INSPECT_PREPARE`

| 字段 | 类型 | 说明 |
|---|---|---|
| `capture_id` | `uint32_t` | 本次检测编号 |
| `need_surface` | `uint8_t` | 是否抓拍表面图 |
| `need_backlight` | `uint8_t` | 是否抓拍常亮背光轮廓图 |
| `need_eddy` | `uint8_t` | 是否采集涡流 |
| `reserved` | `uint8_t` | 保留 |

##### `0x15 CMD_SORT_REQUEST`

| 字段 | 类型 | 说明 |
|---|---|---|
| `capture_id` | `uint32_t` | 对应检测编号 |
| `sort_type` | `uint8_t` | `0`: 不动作，`1`: 良品，`2`: 坏品，`3`: 待复核盒 |
| `grip_recipe` | `uint8_t` | 抓取动作组编号 |
| `part_type` | `uint8_t` | 零件类型编号 |
| `priority` | `uint8_t` | 任务优先级 |

##### `0x16 CMD_MANUAL_AXIS`

| 字段 | 类型 | 说明 |
|---|---|---|
| `device` | `uint8_t` | `0`: 输送带，`1`: 机械臂，`2`: 吸盘/夹爪 |
| `action` | `uint8_t` | 动作编号 |
| `value1` | `int32_t` | 位置/速度/角度参数 |
| `value2` | `int32_t` | 扩展参数 |

##### `0x17 CMD_SET_SURFACE_LIGHT`

由于你的底部背光是**一直开启**，这里仅控制顶部表面补光，不控制背光。

| 字段 | 类型 | 说明 |
|---|---|---|
| `enable` | `uint8_t` | `0`: 关闭，`1`: 开启 |
| `brightness` | `uint8_t` | `0~100` |
| `mode` | `uint8_t` | `0`: 常亮，`1`: 触发增强 |
| `reserved` | `uint8_t` | 保留 |

#### 6.11.6 F4 -> MP157 状态与结果定义

| `msg_id` | 名称 | 作用 |

| `0x34` | `STS_SORT_RESULT` | 分拣动作执行结果 |
| `0x35` | `STS_HOME_RESULT` | 回零结果 |
| `0x36` | `STS_PARAM_RESULT` | 参数同步结果 |

##### `0x30 STS_SYS_STATE`

| 字段 | 类型 | 说明 |

| `auto_mode` | `uint8_t` | 是否自动模式 |

建议 `20 Hz` 上报。

| 字段 | 类型 | 说明 |
|---|---|---|
| `part_present` | `uint8_t` | 光电检测到零件 |

| `limit_x` | `uint8_t` | X 轴限位 |
| `limit_y` | `uint8_t` | Y 轴限位 |

##### `0x33 STS_EDDY_RESULT`

| 字段 | 类型 | 说明 |
|---|---|---|
| `capture_id` | `uint32_t` | 对应检测编号 |
| `amp_x1000` | `int16_t` | 幅值乘 1000 |

##### `0x34 STS_SORT_RESULT`

| 字段 | 类型 | 说明 |
|---|---|---|
| `capture_id` | `uint32_t` | 对应检测编号 |
| `sort_type` | `uint8_t` | 实际执行的分拣类型 |

#### 6.11.7 心跳帧

建议双方都发心跳，便于断线检测。

| 项目 | 建议 |
|---|---|
| MP157 心跳周期 | `500 ms` |
| F4 心跳周期 | `500 ms` |
| 失联判定 | 连续 `3` 个周期未收到对端心跳 |
| 失联处理 | F4 停带 + 禁止分拣；MP157 弹出告警并停止自动模式 |

心跳负载：

| 字段 | 类型 | 说明 |
|---|---|---|
| `uptime_s` | `uint32_t` | 已运行秒数 |
| `cpu_load_x100` | `uint16_t` | MP157 可填 CPU 占用乘 100，F4 固定填 0 |
| `free_mem_kb` | `uint16_t` | MP157 可填剩余内存，F4 固定填 0 |
| `flags` | `uint16_t` | 状态标志位 |

#### 6.11.8 告警码建议

| 错误码 | 含义 |
|---|---|
| `0x0001` | MP157 串口超时 |
| `0x0002` | F4 串口超时 |
| `0x0003` | 相机掉线 |
| `0x0004` | 传送带堵转 |
| `0x0005` | 机械臂原点丢失 |
| `0x0006` | 分拣动作超时 |
| `0x0007` | 急停按下 |
| `0x0008` | 涡流采样异常 |
| `0x0009` | 图像抓拍失败 |

#### 6.11.9 推荐通信节奏

| 消息 | 周期 |
|---|---|
| `CMD_TRACK_OFFSET` | `20~30 Hz` |
| `STS_MOTION` | `20 Hz` |
| `STS_SENSOR` | `10 Hz` |
| `HEARTBEAT` | `2 Hz` |
| `STS_EDDY_RESULT` | 每次检测 1 次 |
| `STS_SORT_RESULT` | 每次分拣结束 1 次 |

#### 6.11.10 一次完整检测的串口时序

```text
MP157 -> F4 : CMD_START_AUTO
F4    -> MP157 : ACK
F4    -> MP157 : STS_SYS_STATE(FEEDING)

MP157 -> F4 : CMD_TRACK_OFFSET(dx_px = 36, target_pulse_delta = 420)
MP157 -> F4 : CMD_TRACK_OFFSET(dx_px = 12, target_pulse_delta = 110)
MP157 -> F4 : CMD_TRACK_OFFSET(dx_px = 4, center_locked = 1, control_level = 2)

MP157 -> F4 : CMD_INSPECT_PREPARE(need_surface=1, need_backlight=1, need_eddy=1)
F4    -> MP157 : ACK
F4    -> MP157 : STS_SYS_STATE(CAPTURE_READY)
F4    -> MP157 : STS_EDDY_RESULT(...)

MP157 本地完成表面检测、背光检测和融合

MP157 -> F4 : CMD_SORT_REQUEST(sort_type = BAD)
F4    -> MP157 : ACK
F4    -> MP157 : STS_SORT_RESULT(done=1, success=1)
F4    -> MP157 : STS_SYS_STATE(FEEDING)
```

## 7. 检测算法完整链路

### 7.1 表面检测

| 项目 | 方案 |
|---|---|
| 图像来源 | 顶部 UVC 摄像头 |
| 目标缺陷 | 划痕、锈蚀、毛刺、局部压伤、表面裂纹 |
| 基础方案 | OpenCV 预处理 + MobileNetV3-Small INT8 |
| 进阶方案 | MobileNetV3-Small + LR-ASPP INT8 输出缺陷掩码 |

### 7.2 背光轮廓检测

| 项目 | 方案 |
|---|---|
| 图像来源 | 底部背光常亮 + 顶部摄像头透射成像 |
| 目标缺陷 | 缺口、孔位异常、断裂、轮廓变形、部分薄厚异常 |
| 推荐方法 | 二值化、轮廓提取、轮廓模板匹配、孔位几何测量 |

### 7.3 涡流检测

| 项目 | 方案 |
|---|---|
| 采样位置 | 零件经过指定线圈探头位置时 |
| 执行芯片 | STM32F4 |
| 目标缺陷 | 近表层裂纹、导电率异常、局部金属状态变化 |
| 推荐方法 | ADC/DMA 采样，提取幅值、相位、峰峰值，阈值判断 |

### 7.4 结果融合

| 输入 | 融合方式 |
|---|---|
| 表面视觉分类结果 | 置信度加权 |
| 背光轮廓异常结果 | 规则权重 |
| 涡流异常结果 | 规则权重 |

推荐输出：

| 输出 | 含义 |
|---|---|
| `GOOD` | 明确正常 |
| `BAD` | 明确异常 |
| `UNCERTAIN` | 模型或多传感器结果冲突，进入人工/云端复核 |

推荐规则示例：

| 条件 | 输出 |
|---|---|
| 表面异常且背光异常 | `BAD` |
| 表面正常但涡流明显异常 | `BAD` |
| 表面异常但背光与涡流正常，且置信度低 | `UNCERTAIN` |
| 三路均正常 | `GOOD` |

### 10.3 检测记录时间字段设计

为了让历史记录、云端追溯和 COS 对账保持一致，检测记录建议至少保存以下时间字段：

| 字段 | 含义 | 是否必须 | 说明 |
|---|---|---|---|
| `captured_at` | 相机拍到用于检测的图片时间 | 必须 | 这是业务上的“检测发生时间” |
| `detected_at` | 模型完成判定的时间 | 建议 | 若流程紧凑，可与 `captured_at` 相同或接近 |
| `uploaded_at` | 图片或结果图上传到云端成功的时间 | 必须 | 表示云端持久化完成时间 |
| `storage_last_modified` | COS 返回的对象最后修改时间 | 可选 | 作为 COS 侧校验字段，不作为主业务时间 |

推荐规则：

| 场景 | 使用字段 |
|---|---|
| 历史记录默认排序 | `captured_at DESC` |
| 云端同步延迟分析 | `uploaded_at - captured_at` |
| COS 对账 | `storage_last_modified` |

说明：

1. 如果 MP157 检测后立即上传，`captured_at` 与 `detected_at` 往往只差几十到几百毫秒。
2. 但 `uploaded_at` 仍需独立保存，因为断网缓存、重试上传和服务器处理都会让它晚于真实检测时间。
3. COS 会记录对象的最后修改时间，但它只能说明“对象何时写入存储”，不能直接代表检测发生时间。

### 10.4 云端 AI 复核逻辑

| 条件 | 处理方式 |
|---|---|
| 本地明确判好 | 直接存档 |
| 本地明确判坏 | 直接存档，可抽检复核 |
| 本地结果冲突或低置信度 | 上传至云端 AI 复核 |
| 人工发现误判 | 手工修正并回写数据库 |

## 12. 关键风险与规避建议

| 风险 | 影响 | 建议 |
|---|---|---|
| 手动对焦 UVC 相机不稳定 | 图像模糊，检测不稳 | 固定拍摄距离，不做升降主方案 |
| 同屏多个零件 | 定位与检测都易出错 | 加挡料与单件节拍控制 |
| 传送带惯性大 | 居中后停不准 | 加减速控制 + 低速点动微调 |
| 模型过重 | 推理慢、界面卡顿 | 只用轻量 INT8 模型 |
| Qt 页面过度复杂 | 响应慢 | 主页面简洁，重点展示实时闭环 |
| 涡流结果噪声大 | 误判 | 做平均滤波和多次采样 |
